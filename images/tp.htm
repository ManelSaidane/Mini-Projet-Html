I. Un peu d'Historique

HTML signifie HyperText Markup Language, soit littéralement "Langage Hypertexte à Balise". HTML a été crée par la communauté scientifique pour pouvoir communiquer les résultats de leurs recherches rapidement via Internet. La contrainte de départ était de pouvoir transférer les documents rapidement, donc partiellement, tout en étant consultable quelle que soit la plate forme informatique cible. Depuis sa naissance les spécifications et possibilités du langage ont beaucoup évoluées.

Remarques :

On parle indifféremment de page WEB, page HTML, document WEB ou encore de fichier HTML.
Sur Internet, chaque page HTML est localisée grâce à un chemin que l'on appelle URL pour "Uniform Resource Location".

II. Format d'une page HTML : balises, structuration d'un document ...

Une page WEB est donc décrite grâce au langage HTML qui sera "compris" par le navigateur Web chargé de l'afficher (hors du contexte du navigateur, une page HTML est simplement un fichier texte structuré). Ce langage découpe chaque document WEB selon un format bien précis dont voici le squelette minimal :

<HTML>
<HEAD>
<TITLE>Ceci est le titre du document</TITLE>
</HEAD>
<BODY>
Ceci est le corps du document
</BODY>
</HTML>

Le code HTML ci-dessus représente un document HTML minimal. Chaque élément entre "<" et ">" s'appelle une balise. Le rôle d'une balise est de délimiter une zone de texte que le navigateur affichera ou interprètera en fonction de la signification de la balise.

Par exemple, lorsque le navigateur rencontre la ligne :
<TITLE>Ceci est le titre du document</TITLE>
il insère la chaine "Ceci est le titre du document" dans la barre de titre de la fenêtre du navigateur. Il est à noter que les balises ne doivent pas se trouver n'importe où, certaines sont imbriquées d'une manière bien précise pour préciser des zones du document. C'est le cas notamment des balises <HEAD>, <BODY> ou encore de celles permettant de définir les lignes et colonnes d'un tableau ( balises <TR> et <TD>). A noter également que dans le cas général chaque balise ouvrante (<xx>) possède une balise fermante (</xx>).

III. Balises "méta"

Dans la zone en-tête du document (délimitée par les balises <HEAD> et </HEAD>), on trouve certaines balises particulières appelées balises META.

Ces dernières ne sont pas affichées par le navigateur mais lui fournissent certaines informations, comme par exemple la langue utilisée. Certaines servent aux moteurs de recherche comme les balises suivantes :

<meta name="author" content="Votre nom Exp: Ettarres Yamna ">
<meta name="reply-to" content="yamna.ettarres@xxxx.xx">
<meta name="description" content="Ma description">
<meta name="keywords" content="HTML, Dreamweaver">
<meta name="generator" content="Macromedia Dreamweaver ">
<meta name="language" content="fr">

D'autres informations dans la partie en-tête permettent de créer des liens entre le document actif et un autre document comme une feuille de style.

Maintenant que vous savez ce qu'est une page HTML, et ce qu'est HTML lui même, cette section montre en quoi DREAMWEAVER intervient pour faciliter la création de pages.

IV. Edition des pages

HTML apparait comme un langage plus ou moins complexe. Aussi s'il est possible d'éditer une page HTML avec un outils aussi simple que le bloc note, il est beaucoup plus pratique d'utiliser des outils dédiés, ces dernier permettant de gagner du temps et d'éviter des erreurs.

DREAMWEAVER est un outils d'édition de page HTML en mode WYSIWYG (What You See Is What You Get). Il permet de créer des pages Web sans connaitre le HTML, tout en visualisant l'aspect de la page en même temps qu'on l'édite. Dans ce principe, il est relativement similaire à un outils comme Word.

DREAMWEAVER ne fait que masquer la complexité et les spécificités du HTML, mais ne les inhibe pas : il en subit forcément les contraintes et les concepts. Il est ainsi important de prendre en compte lors de l'édition des pages, le fait que les documents HTML peuvent être affichés différemment suivant des plate-formes informatiques hétérogènes, avec des navigateurs différents. La première chose à faire lorsqu'on souhaite créer un site WEB est donc de définir une charte graphique qui se présentera correctement sur tout environnement cible. Pour cette même raison, il est important de tester les pages sur au moins les deux navigateurs standards actuels : Netscape Navigator et Internet Explorer (IE).

V. Gestion du site

A partir du moment où une page HTML n'est plus seule et isolée, on parle de site WEB. La difficulté de gestion d'un site WEB consiste en la gestion des différentes pages et fichiers annexes (images ...), et à la mise à jour des liens hypertextes car on se retrouve très rapidement avec une arborescence imposante de fichiers, certains étant à jour sur le serveur alors que d'autres sont en cours de modification ou de création.

Heureusement, un outils comme DREAMWEAVER (mais ce n'est bien sûr pas le seul ...) permet outre l'édition de pages HTML, la gestion d'un site. En effet il permet : Il est vrai que FrontPage ou Visual Interdev par exemple, remplissent très bien ce rôle, il en reste néammoins que le serveur doit être de la gamme Microsoft (NT4 ou 2000 Serveur) avec IIS et de surcroit les extensions FrontPage... Le choix ici dépend de la politique que l'on souhaite adopter ...
